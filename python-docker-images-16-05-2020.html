docker образы python

вчера поэкспериментировал с докеровскими образами питона.

есть стандартный образ python:3.8 условно, с ним образ приложения весит 1Гиг бля.

попробовал python:3.8-alpine — сам образ питона аля 60Метров, но сборка не взлетела, в нём не хватает линуксовых завимостей, то есть когда начинаются ставиться питоновские сторонние пакеты для приложухи, то всё крашится, потому-что нет каких-то линуксовых зависимостей в этом образе, а ставить их тоже затея хз, они могут за собой потянуть ещё и ещё на таком маленьком образе.

в итоге попробовал python:3.8-slim и всё взлетело без лишнего пука — вес образа с приложением 294Метра, собирается на ssd за 15сек.

короче мораль такая

если в проекте используется только стандартная питоновская библиотека или обертки над ней, то можно юзать alpine.

если есть какие-то зависимости, но их не так чтобы до хуя, то можно попробовать slim.

если крупный проект с овердохуя зависимостями, то стандартный образ.
